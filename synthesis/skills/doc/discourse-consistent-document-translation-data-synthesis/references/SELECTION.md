# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics:**
  * **The source passage must contain a genuine discourse dependency.** Accept only if at least one translation decision depends on earlier or later context, such as a zero pronoun, referential ambiguity, lexical consistency need, or discourse deixis. This is the first gate for the skill. If every sentence can be translated correctly in isolation, the example is not sufficiently diagnostic.
  * **The trajectory must document the dependency resolution.** The path should show how the agent used context to choose the final phrasing, reference, or style. This keeps the synthetic data behaviorally meaningful. If the context-sensitive choice is hidden, the example loses much of its training value.
  * **Global consistency must be auditable.** A reviewer should be able to scan the final answer and verify that entity reference, terminology, and voice remain stable across sentences. This is the practical evaluation target. If the output is fluent but inconsistently anchored, reject it.
  * **The source should stress more than one discourse cue when possible.** Strong examples often combine a referential ambiguity with a lexical-cohesion requirement or a style requirement with an omitted subject. This produces richer supervision. A single-cue example is still acceptable, but it should be a sharp one.
* **Rejection Criteria:**
  * **Reject sentence-level solvable passages.** Discard any source where an isolated sentence translator would perform equally well because no cross-sentence cue matters. Those instances do not belong in this skill. Fluency alone is not enough.
  * **Reject purely lexical trap questions.** If the example only checks a rare-word lookup and not discourse consistency, it should be removed. This skill targets document context, not vocabulary difficulty. An obscure noun without cross-sentence dependency is not sufficient.
  * **Reject unstable references.** If even a careful reader cannot tell who a pronoun or ellipsis refers to from the source context, the instance becomes annotation noise rather than useful difficulty. The benchmark should contain resolvable ambiguity, not irreducible ambiguity. In practice, discard passages where two discourse interpretations remain equally plausible.
  * **Reject outputs that are good locally but broken globally.** A selected trajectory should not tolerate a final answer that changes narrator, speaker, or terminology mid-passage without justification. Those are precisely the failure modes this skill exists to expose. If the answer reads like a set of independent subtitle lines, reject it.
