# Phase 1: Environment Exploration Guide
* **Exploration Strategy:**
  * **Segment the source by discourse role, not only by token length.** During exploration, note where the source introduces actors, shifts topics, resolves earlier questions, or changes time, perspective, or section function. This creates a discourse map that survives later chunking and recompression. An edge case is a long report where an appendix looks low-salience globally but contains the definition needed to make earlier claims coherent.
  * **Track running entities and unresolved references.** Keep a compact state list of the major entities, themes, obligations, or plot lines that persist across chunks. This prevents later summaries from referring to a person, topic, or decision without enough context. A common example is a running summary that says “he,” “the company,” or “the proposal” long after the original anchor was lost.
  * **Mark locally salient material that is globally low value.** Some chunks contain dramatic but ultimately non-central details that can distort the summary if over-preserved. Exploration should therefore separate local excitement from global narrative importance. An edge case is a vivid anecdote in a memoir or a detailed exception note in a report that is memorable but not structurally central.
  * **Record inter-chunk dependencies explicitly.** When one chunk only becomes understandable because of another, note that dependency in the ledger before summarization begins. This helps the agent later avoid discontinuity, inconsistency, and causal omission. A concrete example is marking that an early mention of an investigation is only explained several chapters or sections later.
* **Target Trajectory Profile:**
  * **A good trajectory preserves global continuity.** The path should make clear how chunk-level observations were woven into one readable line of development rather than dumped as isolated facts. This is the defining property of long-form summarization quality. If the generated summary could be permuted sentence-by-sentence without changing meaning, the trajectory is not coherent enough.
  * **A good trajectory handles compression visibly.** The path should reveal when information was merged, when details were dropped, and why those decisions were made. This is important because chunk-and-compress pipelines fail in exactly those moments. An edge case is a detail removed for brevity that later turns out to be required for understanding a causal link.
  * **A good trajectory minimizes omission and duplication simultaneously.** Long-form summaries often fail by dropping necessary context or by repeating similar information after multiple merges. A strong trajectory shows that both risks were actively managed. If the summary includes one but not the other, it should still be scrutinized as a weak candidate.
  * **A good trajectory ends in a self-sufficient summary.** A reader who never saw the source should still be able to follow the generated summary without asking “who is this,” “why did this happen,” or “why did the narrative jump here.” This self-sufficiency is the point of the skill. If the summary only works as a memory aid for someone who already knows the source, the example is not strong enough.
