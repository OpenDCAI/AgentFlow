# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics:**
  * **Evidence sufficiency must be explicit.** Accept the trajectory only if it contains every evidence unit needed to justify the answer, with no silent jumps across missing spans. For this capability that usually means one decisive grounded span, or one decisive page region in a visual document. If the final answer mentions an entity, year, or value that never appeared in the trajectory, reject it immediately.
  * **The trajectory must reflect the intended capability rather than a shortcut.** Accept only if the solution truly depends on exact local evidence recovery inside long or visually rich documents, not on a trivial keyword match or prior world knowledge. This can be checked by asking whether a human without access to the located evidence could still answer confidently. If the answer is yes, the example is not diagnostic enough.
  * **Disambiguation quality must be verifiable.** Accept only when the trajectory either resolves a realistic ambiguity or demonstrates why an apparently plausible alternative is wrong. This guards against synthetic data that is clean but unrealistic. A concrete check is whether at least one distractor with the same vocabulary, same metric family, or same visual form was available and successfully ruled out.
  * **The final answer must be reconstructible from the path.** The evaluator should be able to derive the final answer by following the trajectory in order and applying only minimal reasoning. This keeps the example auditable and prevents hidden leaps. If a separate explanation would be required to understand how the answer emerged, the trajectory is underspecified.
* **Rejection Criteria:**
  * **Reject trivial locality failures.** Discard trajectories where the answer can be copied from the first obvious hit without any meaningful filtering, grounding, or constraint checking. Such examples do not stress a real document agent because they reward shallow keyword spotting. A common edge case is a section title that exactly restates the question and contains the answer in the first line.
  * **Reject under-specified evidence.** Discard any trajectory where the cited evidence is too vague, too broad, or too paraphrased to support the claimed answer. A whole page, whole document, or whole table is not enough if the decisive region is never localized. This is especially important when multiple near-duplicate values or entities appear in the same neighborhood.
  * **Reject capability mismatch.** If the example is supposedly about evidence-localized QA but is actually solved by a simpler skill, discard it. For instance, a question that can be answered from the title page, or one that only requires generic background knowledge, should not pass. This prevents contamination with easier retrieval or trivia behavior.
  * **Reject noisy or unstable synthesis targets.** Throw away examples where OCR breakage, unreadable chart axes, broken page order, or missing layout context would make scoring unreliable. The point of this skill is to stress grounding, not accidental corruption. An edge case is a slide screenshot whose text is too blurred for independent verification.
