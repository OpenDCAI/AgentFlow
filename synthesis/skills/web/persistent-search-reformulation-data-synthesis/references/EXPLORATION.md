# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - Begin from the most oblique clue bundle rather than from the expected answer type. First issue one broad retrieval query that exposes the search landscape, then inspect whether the returned pages are organized by event, person, venue, publication, or time window. If the first query already exposes the answer verbatim, discard that seed and restart with a harder one because this capability requires visible reformulation pressure.
  - Rephrase every failed or low-yield search using a different semantic axis, not just a slightly edited keyword string. Move systematically between entity-centric wording, role-centric wording, date-bounded wording, and reverse-lookup wording so that each iteration tests a genuinely different retrieval hypothesis. For example, if “EMNLP Dartmouth Penn paper” is too broad, pivot to author education pages first and only later come back to EMNLP proceedings with narrowed candidates.
  - Track why each search step failed before issuing the next one. Distinguish between no-hit failure, too-many-hit failure, misleading-hit failure, and stale-hit failure, because each failure mode implies a different next action. As an edge case, if the first page of results is dominated by summaries or AI snippets, the agent should deliberately open source pages and mine discriminating terms before reformulating again.
  - Favor trajectories that expose at least one recoverable dead end. A good exploration path should show the agent committing to an initially plausible direction, discovering that the evidence is incomplete or contradictory, and then pivoting with a justified query rewrite. For example, an agent might start from referee records, realize many matches fit the nationality clue, and then refine around card timing and injury substitution constraints.

* **Target Trajectory Profile**:
  - A target trajectory should contain multiple materially different search formulations, not cosmetic variants. At minimum, the agent should demonstrate a broad probe, one or more narrowing turns, and a final disambiguation turn that closes the search space. If all queries share the same wording skeleton, the trajectory is too shallow for this capability.
  - The trajectory should expose progressive information gain at each step. Every reformulation must eliminate a class of candidates, add a new discriminative clue, or shift to a better document type such as official pages, archival results, or structured indexes. An edge case to watch for is accidental answer leakage from snippets; if that happens, the agent should still verify the answer through a reformulated evidence-seeking path.
  - A strong trajectory includes at least one explicit backtrack. The agent should recognize when a promising query path produces near misses, record what was learned, and then redirect the search rather than repeating the same pattern. For example, if searching by team statistics gives too many matches, the agent should pivot to referee or substitution logs instead of adding more generic football terms.
  - The final evidence chain should only become apparent after reformulation, not before it. The path should make it clear that the answer could not have been obtained reliably from the starting query alone. As a concrete edge case, if a candidate answer appears early, the trajectory still needs a subsequent query or navigation step that proves why competing candidates fail.
