# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - Identify early whether the task lives inside a language-specific web ecosystem. The agent should inspect whether the entities, institutions, or cultural artifacts are primarily indexed under non-English names, then pivot its query language accordingly. An edge case is bilingual leakage from English summaries; these can be useful starting points, but they should not replace native-source verification.
  - Generate and test multiple name variants before browsing deeply. This includes translated names, transliterations, native-script names, abbreviated forms, and domain-specific aliases. For example, a cultural artifact may be described differently in English prose than in the wording used on heritage-list pages.
  - Use ecosystem-appropriate search surfaces rather than assuming one global search engine is sufficient. Site-specific search, local portals, regional encyclopedic pages, and government listings may be far more productive than general English web search. If a broad search engine keeps surfacing translated commentary instead of primary local pages, the agent should pivot to native-language or site-specific retrieval.
  - Preserve the original-language evidence even if the final answer is returned in another language. The agent should record which non-English pages resolved the decisive clues and how the translation or transliteration was normalized. As an edge case, if two English renderings compete, the native-language page should determine which mapping is correct.

* **Target Trajectory Profile**:
  - A target trajectory should include at least one decisive non-English source. The crucial clue cannot be obtainable solely from English-language mirrors or summaries. If every important step can be solved in English, the example does not truly test cross-lingual navigation.
  - The trajectory should show explicit handling of naming variation. The agent must move between at least two surface forms of the same entity, such as English translation and native script. Without visible name normalization, the browsing path is not sufficiently cross-lingual.
  - A strong trajectory bridges retrieval and interpretation. It should not merely open a native-language page and copy text; it should connect that page to the original prompt constraints and explain why the retrieved term answers the question. The edge case to avoid is blind translation without evidential grounding.
  - The final synthesis should preserve semantic fidelity across languages. If the answer has multiple acceptable English renderings, the agent should choose the one best aligned with the environment’s evidence and the benchmark’s grading rules. A trajectory that loses this alignment during translation is not acceptable.
