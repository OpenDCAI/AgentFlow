# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics**:
  - **Conflict or unsupported drift is objectively verifiable.** Accept only when a reviewer can point to the exact contradiction or unsupported extension in the corpus or generated claim. The flaw should not depend on speculative outside interpretation. This makes the sample reliable for automated evaluation.
  - **A faithful correction path exists.** Accept only if the agent can respond correctly by either correcting, narrowing, or abstaining based on the retrieved evidence pattern. If no defensible grounded response exists, the sample belongs in a pure rejection skill instead. Conflict-aware grounding assumes some principled response is available.
  - **The main challenge is fidelity, not retrieval breadth.** Accept when the central difficulty comes from staying faithful under conflict or semantic drift, not from finding one more obscure document. Retrieval may still matter, but it should not dominate the sample. Otherwise the task leaks into another capability.
  - **The error pressure is realistic.** Keep cases where the wrong answer would sound fluent and plausible to a human reader. This is important because practical hallucination failures are often convincing rather than obviously absurd. Stylized nonsense is not suitable training material here.
* **Rejection Criteria**:
  - **No identifiable support boundary.** Reject if annotators cannot clearly mark where support ends and hallucination begins. Without that boundary, the example cannot supervise faithful grounding precisely. Ambiguous gray zones should be removed or rewritten.
  - **Conflict requires specialist outside knowledge.** Reject if resolving the contradiction depends on domain knowledge not present in the environment and not granted by the benchmark setup. This would make the task unfair for a bounded-corpus RAG agent. Prefer self-contained or warning-conditioned contradictions.
  - **All evidence is equally broken.** Reject if every available source is corrupted and no grounded correction or bounded abstention is possible. Such samples do not teach selective faithfulness. They often devolve into arbitrary model priors.
  - **Generated target itself contains multiple unrelated errors.** Reject examples where many different failure modes overlap at once and obscure the intended supervision signal. Keep each sample centered on one dominant conflict or one dominant unsupported extension. This preserves interpretability.
