# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - **Seed explicit conflicts and subtle drifts.** Explore both hard contradictions, such as wrong entities or numbers, and softer semantic drifts, such as changing severity, range meaning, or causal force. Create candidate bundles where the conflict is verifiable from the provided context and does not require hidden domain expertise to notice. A good edge case is when every token is locally plausible but the composed claim is unsupported.
  - **Separate supported facts from tempting continuations.** During exploration, identify where the corpus truly stops supporting a claim and where a fluent model is likely to continue by habit. Write down the last fully supported proposition before any unsupported extension begins. This boundary is the key object the downstream skill must learn to respect.
  - **Capture contradiction provenance.** When documents disagree, note which chunk carries the conflict, which chunk carries the corrective signal, and whether the correction is internal to the corpus or only externally known. Prefer cases where the corpus itself enables detection or where the benchmark explicitly grants warning-based correction behavior. This keeps the synthetic task auditable and reproducible.
  - **Test correction without overcorrection.** Explore variants where the model could respond by overreacting, such as declaring all evidence unusable even though a supported narrow answer is still available. Keep examples where the correct behavior is selective correction, not blanket refusal. That distinction is important for practical RAG use.
* **Target Trajectory Profile**:
  - **Claim-level checking.** A good trajectory breaks the response into atomic or near-atomic claims and checks each against retrieved evidence. The agent should verify numbers, entities, ranges, and relations separately before composing the final answer. This prevents one supported fragment from laundering an unsupported whole sentence.
  - **Conflict tagging before synthesis.** The trace should explicitly tag contradicted chunks or unsupported expansions before generating the final response. This ordering matters because once a fluent answer is written, unsupported details are harder to notice. Conflict tagging acts as a guardrail against contamination.
  - **Selective preservation of still-valid content.** When only part of a candidate answer is flawed, the trajectory should preserve the supported parts and discard only the unsupported or contradicted fragment. Do not collapse everything into a refusal unless the conflict destroys the whole answer. This models the practical need for precise correction.
  - **Final answer stays within the support envelope.** The answer should use wording that is no stronger, broader, or more specific than the evidence permits. If the corpus supports a narrow interval, keep the interval narrow; if it supports uncertainty, keep the uncertainty visible. Support-envelope discipline is the essence of this skill.
