# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - **Index both nodes and edges as retrievable units.** During exploration, treat node attributes and edge attributes as separate yet coordinated evidence objects. This allows the agent to find not only relevant entities but also the relations that make the answer derivable. Graph QA fails if relations are not first-class retrieval targets.
  - **Retrieve for connectivity, not only similarity.** After initial query-node or query-edge matching, explore whether the retrieved pieces form a connected reasoning subgraph. If not, look for bridge nodes or edges that restore connectivity while staying query relevant. A disconnected evidence bag is usually insufficient for graph reasoning.
  - **Stress-test against graph hallucination.** Construct cases where a language model could easily invent a missing node, relation, or path if the retrieved subgraph is incomplete. Then verify whether the exploration trace can anchor every claimed relation to an actual graph element. Graph settings are especially vulnerable to invented structure.
  - **Use multiple graph modalities.** Explore explanation graphs, scene graphs, and knowledge graphs so the skill covers argumentative, spatial, and factual relation patterns. This broadens the synthetic dataset beyond one graph domain. A reusable graph skill should transfer across structured environments.
* **Target Trajectory Profile**:
  - **Query-conditioned node and edge retrieval.** A good trajectory first identifies the most relevant nodes and edges for the current question. The trace should show why each selected graph element matters to the query. This is the graph analogue of evidence selection in text RAG.
  - **Subgraph construction is explicit.** Do not jump directly from scattered nodes and edges to the answer; show how they form a connected or interpretable subgraph. The connecting structure is often the actual reasoning object. Explicit subgraph construction also improves explainability.
  - **Path or relation reasoning is stated.** The trajectory should name the relation composition or path traversal used, such as spatial chaining, stance relation resolution, or family-link traversal. This makes the reasoning inspectable and reduces hidden hallucination. Graph QA succeeds when relation use is explicit.
  - **Answer cites graph elements, not free prose alone.** The final response should point back to supporting nodes or edges in concise form. This is particularly useful for preventing invented nodes or edges from slipping into the answer. Element-grounded answers are the most reliable supervision signal here.
