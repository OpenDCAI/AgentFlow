# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics**:
  - **Decisive support is absent everywhere inspected.** Accept only when the trace shows that the required answer field does not appear in any plausible supporting document. The agent must have examined enough relevant candidates to justify that claim. If one overlooked chunk actually answers the question, the sample is invalid.
  - **Topical relevance remains high.** Accept only if the retrieved set still looks relevant enough that an overconfident model could guess. This ensures the task measures rejection under pressure, not rejection on obviously unrelated inputs. A good bundle should tempt but not support.
  - **Refusal is the unique correct target.** The accepted case must have no alternate grounded answer, partial answer, or safe narrowing that would also count as correct. The right behavior should be a clear abstention, not a hedged content answer. This keeps evaluation crisp.
  - **Trace demonstrates boundedness.** Accept when the refusal follows explicit corpus inspection and not mere uncertainty language. A reviewer should be able to see why the answer is absent from the environment. This distinguishes genuine evidence-bounded rejection from stylistic caution.
* **Rejection Criteria**:
  - **Refusal can be bypassed by one obvious chunk.** Reject if one document actually contains the answer or nearly all of it. Such a case teaches retrieval quality, not evidence-missing rejection. Inspect borderline passages carefully before keeping the sample.
  - **Question is underspecified rather than unsupported.** Reject if the issue is that the query itself is ambiguous, malformed, or missing a necessary qualifier. This skill is about absent evidence, not bad question writing. The question should be perfectly interpretable even though the corpus cannot answer it.
  - **Bundle is obviously off-topic.** Reject if the retrieval set is so unrelated that any model would refuse instantly. That produces easy abstention and fails to simulate realistic RAG failure modes. Keep only corpora that look plausibly useful at first glance.
  - **Response mixes guessing with abstention.** Reject trajectories where the model states a likely answer and then adds a soft caveat. Those outputs are hard to score and teach the wrong behavior. A clean refusal signal is required for this skill.
