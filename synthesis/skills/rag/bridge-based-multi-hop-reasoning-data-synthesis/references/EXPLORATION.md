# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - **Identify candidate bridge entities or topics.** During exploration, mine the corpus for repeated entities, concepts, or relation labels that can connect otherwise separate documents. Record each candidate bridge together with the documents it links and the distinct claim each document contributes. A bridge is useful only if it enables a reasoning chain rather than mere topical clustering.
  - **Force at least one hop boundary.** Construct exploration traces where the answer cannot be read directly from a single document and instead requires crossing at least one bridge boundary. Verify this by attempting single-document answering and keeping only cases that still fail. This protects the skill from collapsing into ordinary extraction.
  - **Keep the bridge explicit.** Make the shared bridge visible in the trace, either as an entity, topic, relation, or common attribute. When possible, preserve the bridge term verbatim so later reviewers can audit the hop. Hidden or implicit bridges make training signals too opaque.
  - **Probe bridge distractors.** Add at least one distractor bridge candidate that is lexically plausible but leads to the wrong chain, then observe whether the agent can avoid it. This mirrors realistic retrieval where many shared entities exist but only one supports the intended reasoning path. A strong multi-hop sample is fragile to wrong-bridge selection.
* **Target Trajectory Profile**:
  - **Bridge declaration before inference.** A good trajectory names the bridge first and then shows how each document contributes a claim anchored to that bridge. This sequencing makes the reasoning chain inspectable. Without an explicit bridge declaration, the answer often looks like a lucky jump.
  - **Per-hop evidence retention.** After each hop, keep the intermediate conclusion in the trajectory so the final inference has a visible foundation. Do not compress multiple hops into one broad paraphrase. Intermediate retention is what distinguishes multi-hop reasoning from vague synthesis.
  - **Wrong-bridge elimination.** The trajectory should briefly note why competing bridge candidates are rejected when they appear in the retrieved set. This trains the downstream agent to treat bridge selection as an evidential decision. A missing elimination step often causes brittle behavior.
  - **Final inference comes last.** Only after the bridge path is fully constructed should the agent produce the final yes/no, entity, or computed answer. Premature answering usually indicates that one hop was skipped or assumed. The final step should read like the consequence of an already-built chain.
