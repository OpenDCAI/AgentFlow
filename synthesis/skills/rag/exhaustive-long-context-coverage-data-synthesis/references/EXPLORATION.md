# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - **Map required evidence by document.** During exploration, create a coverage map that states what role each document plays: decisive fact, comparison candidate, grouping member, or contextual distractor. This map should exist before answer synthesis begins. Without a document-level map, long-context tasks quickly become opaque.
  - **Control evidence dispersion deliberately.** Spread required facts across documents so that the answer cannot be trusted unless the trajectory records broad enough coverage. Use same-domain documents with similar format to prevent easy visual shortcuts. High evidence dispersion is the main difficulty driver for this skill.
  - **Include both localization and all-document variants.** Explore some tasks where only one document contains the target fact and others where every candidate document matters to the answer. Keeping both variants makes the skill useful for a broader range of long-context RAG workloads. Localization is the base case; exhaustive aggregation is the full case.
  - **Track omission risk explicitly.** For each trajectory, note which document omissions would change the answer and how. This omission analysis is the quickest way to tell whether the task truly measures coverage. A good sample has at least one omission that predictably flips the result.
* **Target Trajectory Profile**:
  - **Document ledger before conclusion.** A good trajectory keeps a ledger of inspected documents and the extracted contribution of each. The ledger can be terse, but it must be complete enough to show coverage. This prevents silent shortcut answering.
  - **Coverage then operation.** The agent should finish the necessary document sweep before comparing, grouping, or summarizing across the bundle. If it starts concluding after two or three documents when many remain relevant, the trajectory is weak. Long-context coverage requires disciplined postponement of closure.
  - **Uniform treatment of same-format documents.** When documents share a template such as reports or cases, the trace should apply a consistent extraction procedure to each. This reduces arbitrary skipping and makes later auditing easier. A stable per-document procedure is critical in long-context settings.
  - **Final answer references the coverage scope.** The response should reflect that the answer was derived from a defined candidate set rather than from one lucky excerpt. This is especially important for extremum and grouping questions. Coverage scope should remain visible in the reasoning trace even if the final answer is short.
