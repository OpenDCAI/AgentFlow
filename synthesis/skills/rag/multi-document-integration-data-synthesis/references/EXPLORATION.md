# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - **Split the answer into complementary slots.** Design the environment so different documents each contribute one necessary field, event, or constraint needed for the final answer. During exploration, explicitly label which slot each candidate passage fills and confirm that the union is sufficient while each part alone is not. This slot view keeps the task centered on integration rather than retrieval volume.
  - **Preserve non-redundancy between sources.** Choose documents that add genuinely new information rather than paraphrasing each other. If two passages merely restate the same fact, replace one with a document that fills a different role in the reasoning chain. Non-redundancy is what makes integration measurable.
  - **Record the merge operation.** As the agent explores, note whether the final synthesis is conjunction, tuple assembly, comparison-ready alignment, or set completion. The later data synthesis should inherit that merge pattern explicitly. Without a named merge operation, the trajectory often becomes hand-wavy.
  - **Introduce realistic cross-source variation.** Allow the contributing documents to differ in phrasing, granularity, and document style while still aligning on the underlying answer. This prevents the task from being solved by exact string matching across duplicated templates. A useful edge case is when one source gives a full date and another gives only a month-day phrase that still resolves cleanly.
* **Target Trajectory Profile**:
  - **Source-by-source slot filling.** A good trajectory extracts the needed slot from each source separately before attempting to write the final answer. Do not let the agent jump directly to synthesis without showing the partial fills. The partial fills are the proof that multi-document integration actually occurred.
  - **One source, one contribution discipline.** Each cited document in the trajectory should have a clear role in the answer and should not be included only because it is topically related. This keeps the context compact and makes evaluation clean. A source without a unique contribution should be dropped.
  - **Synthesis after coverage, not before.** The answer should be generated only after all mandatory slots or constraints have been collected. If the agent writes an early partial answer and patches it later, record that as a weak trajectory unless the patch is explicit and correct. Integration tasks reward complete evidence coverage before answer emission.
  - **Final response preserves structure.** The final answer should reflect the integrated structure clearly, for example as an ordered pair, coordinated phrase, or concise sentence containing all required parts. Avoid narratives that bury one of the integrated facts. The output form should make missing slots obvious to a reviewer.
