# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - Determine the reporting contract before generating prose. Identify which artifacts must be saved, which chart type is required or most defensible, and whether the final output should be bullet points, a PDF report, or another structured summary. This prevents the common mismatch where the analysis is correct but the presentation format is unusable.
  - Ground every intended insight in a visible numerical or visual anchor. During exploration, decide which values, trends, comparisons, or categories each narrative point will reference. An edge case is a model that writes plausible-sounding insights unsupported by the extracted data or chart.
  - Keep the visualization faithful to the analytic goal. Choose a chart only after confirming the variables, grouping, and relationship type that need to be shown. For example, correlation-oriented tasks may need a scatter plot, whereas categorical comparisons may need grouped bars or stacked views.
  - Plan for artifact reproducibility. The trajectory should note file names, save paths, and any compact data export needed to support later inspection of the figure or report. This is especially important when the insight text is generated from extracted data rather than directly from the image.
* **Target Trajectory Profile**:
  - A good trajectory should produce both an analytical artifact and a communication artifact. Typical pairs are figure plus bullet insights, or metric table plus PDF report. This duality is what distinguishes reporting capability from pure analysis.
  - The written summary should be traceably grounded. A reviewer should be able to link each major claim to a value in data.txt, a figure element, or an explicit computed metric. This is crucial because presentation tasks are vulnerable to confident hallucination.
  - The visualization choice should match the question structure. The trajectory should reveal why a given plot type, grouping, or axis mapping was used and how it supports the requested insight. An edge case to avoid is an attractive but semantically irrelevant chart.
  - The final report should remain concise and decision-useful. Good reporting trajectories extract the minimum set of high-value insights needed by the user. If the output turns into a long generic essay, the capability signal weakens.
