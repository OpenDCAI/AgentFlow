# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - **Build topically relevant but empty evidence packs.** Create retrieval bundles that clearly belong to the same domain as the question while withholding the decisive answer-bearing fact. Use same-entity-family, same-event-family, or same-document-template passages so the absence is non-trivial rather than obvious. For example, adjacent-year award reports or neighboring-company annual reports are better than unrelated filler.
  - **Probe different absence mechanisms.** Explore several ways evidence can be missing: the entity is absent, the asked year is absent, the required relation is absent, or the corpus only contains partial fields. Record which absence mechanism is active for each trajectory so the later synthesis can label insufficiency precisely. This helps avoid creating samples that merely look empty for accidental reasons.
  - **Track exhaustive failure.** A good exploration trace should show that the agent checked the most plausible candidate passages before abstaining. Do this by logging the documents inspected, the fields sought, and the reason each failed to answer the question. The point is to demonstrate grounded rejection, not lazy refusal.
  - **Stress-test against plausible guessing.** Choose question forms where the model could easily guess a fluent answer from world knowledge, priors, or neighboring records. Keep those cases only when the corpus still lacks the decisive support. This edge case is essential because real RAG failures often come from overconfident guessing under topical familiarity.
* **Target Trajectory Profile**:
  - **Search broadly before refusing.** A good trajectory explores the most likely supporting passages and only then produces an insufficiency judgment. The trace should show an active search for evidence rather than an immediate generic refusal. This makes the synthetic data teach informed abstention instead of indiscriminate non-cooperation.
  - **Explicit absence rationale.** The trajectory should say what is missing, such as the target year, target entity, or needed cross-document link. That explanation should be tied to inspected corpus content, not to a vague claim that the model is uncertain. An explicit absence note makes the rejection auditable.
  - **No hidden-world override.** The trace must not rescue the answer using prior model knowledge, commonsense interpolation, or neighboring-document substitution. Any tempting external answer should be acknowledged as unsupported if it is not present in the corpus. This preserves the strict evidence-bounded nature of the task.
  - **Stable refusal wording.** The final step should normalize refusals into a clearly scorable answer pattern such as 'insufficient information'. Keep the wording direct and avoid mixing partial guesses with abstention. A mixed response weakens the supervision signal for refusal behavior.
