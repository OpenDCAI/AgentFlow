# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics**:
  - **At least one decisive passage exists.** Accept only trajectories where one passage contains the exact answer-bearing field needed to resolve the question. The trace must point to that decisive span and show why the distractors fail to answer the same slot. If all passages are merely suggestive, the sample belongs to rejection-oriented synthesis instead.
  - **Distractors are genuinely confusable.** Accept only when distractors overlap strongly with the target topic and would plausibly mislead a shallow model. Check that they share entity family, event type, document format, or temporal neighborhood with the target. Unrelated filler does not create the intended capability stress.
  - **Answer can be justified without extra-world knowledge.** The accepted trajectory must derive the answer solely from the retained decisive evidence. The agent may use reasoning to compare passages, but it must not import hidden knowledge to override the corpus. This keeps the sample faithful to the bounded-corpus RAG setting.
  - **Failure pressure is precision-centric.** Accept the trajectory only if the main risk is selecting the wrong supporting passage, not doing arithmetic or external reasoning. The core observable should be evidence selection under noise. If a different capability dominates the difficulty, move the sample to another skill.
* **Rejection Criteria**:
  - **Noise is too weak.** Reject trajectories where distractors are obviously irrelevant after a one-line skim. If the wrong passages do not look usable, the sample will not actually measure robustness to topical noise. Generic filler paragraphs are therefore disallowed.
  - **Answer is duplicated everywhere.** Reject if multiple passages state the same decisive answer and the task no longer requires discrimination. The point is to separate one decisive chunk from near misses, not to reward redundancy. Otherwise the problem degenerates into recall only.
  - **Missing discriminating metadata.** Reject if the trace cannot verify the decisive contrast because the supporting and distracting passages lack explicit labels such as dates, entity names, or metric headers. Without the visible discriminator, annotation becomes unstable. A reviewer should be able to point to the precise token that settles the case.
  - **Reasoning burden shifts away from evidence filtering.** Reject if solving the sample mostly requires arithmetic, cross-document composition, or table operations beyond simple evidence selection. Those belong to other synthesis skills and would blur the intended signal. Keep this skill centered on precision under confusable retrieval noise.
