# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics:**
  * **The source must materially exceed ordinary context.** Accept the trajectory only if the document is long enough that chunking, merging, or iterative updating is genuinely required. Otherwise the example does not stress long-form coherence. A simple threshold check is whether a one-shot read would have been practical for the base agent.
  * **Coherence pressure must be real.** The source should contain long-distance references, delayed explanations, topic shifts, or recurring entities that make continuity non-trivial. This gives the summarizer something meaningful to preserve. If the source is just a flat list of independent facts, the example is too easy.
  * **The trajectory must show discourse-aware merge decisions.** Accept only if the path records how partial summaries were combined, compressed, or revised with continuity in mind. This makes the data valuable for training real agents rather than only for scoring final text. If the merge steps are invisible, the trajectory is incomplete.
  * **The final summary must be readable on its own.** A selector should reject trajectories whose outputs still contain entity omission, event omission, causal omission, discontinuity, inconsistency, or duplication severe enough to confuse a fresh reader. This mirrors the real evaluation target of the long-summary benchmarks. A concise summary is not acceptable if it is incoherent.
* **Rejection Criteria:**
  * **Reject short-context impostors.** If the source can be summarized comfortably in one pass, the example should not enter this skill. Those cases belong to generic summarization, not long-form coherence. Length alone is not enough; the document must induce true long-range dependency pressure.
  * **Reject chunk-independent sources.** Discard documents where each chunk can be summarized independently without harming the final output. Such documents do not reveal the core failure modes of omission, discontinuity, or contradiction across chunk boundaries. A common example is a collection of loosely related bullet lists.
  * **Reject summaries that are only detail-heavy.** A trajectory should not pass merely because it retains many facts. Dense but incoherent output is a failure mode, not a success. If the summary reads like a stitched list of chunk notes, discard it.
  * **Reject examples with unverifiable merge logic.** If no one can tell why the system kept, dropped, or merged certain details, the synthetic example will be hard to analyze and hard to reuse. The selection stage must therefore prefer paths with explicit compression decisions. Hidden merge heuristics are grounds for rejection.
