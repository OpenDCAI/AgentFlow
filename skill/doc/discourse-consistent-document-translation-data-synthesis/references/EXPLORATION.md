# Phase 1: Environment Exploration Guide
* **Exploration Strategy:**
  * **Read the full discourse unit before translating.** Do not begin from sentence 1 and commit permanently sentence by sentence. Instead, inspect the whole paragraph or the whole document segment first so that speaker identity, narrative stance, omitted arguments, and lexical patterns are visible. An edge case is a pronoun-dropped language where the subject must be inferred from an earlier sentence.
  * **Build a discourse ledger.** Record core entities, who is speaking, tense anchors, deixis, repeated terminology, and any ellipses that must be resolved from context. This gives the translation stage an explicit memory of what must remain consistent. A common example is a retail-dialogue paragraph where “I” and “he” must not be swapped even though local sentences are ambiguous.
  * **Mark terms that should remain lexically stable.** Some passages depend on reusing the same expression or on preserving contrast between near-synonyms. Exploration should therefore identify the terms whose variation would damage cohesion. An edge case is translating a technical document where one entity label should not be paraphrased differently every time it appears.
  * **Surface style-sensitive regions early.** Literary, persuasive, and dialogic passages often depend on register, rhythm, or voice that cannot be repaired after a literal first pass. The exploration phase should mark those stretches before translation begins. A concrete example is a short reply that should sound brisk and natural rather than formally literal.
* **Target Trajectory Profile:**
  * **A good trajectory makes context use visible.** The path should show where the agent looked back to resolve a pronoun, confirm a speaker, or maintain a repeated term. This is essential because document-level translation is defined by using context, not just by producing fluent target sentences. If the path never references cross-sentence context, it is not strong enough.
  * **A good trajectory separates local adequacy from discourse adequacy.** It should be clear that the translation is not only individually sensible sentence by sentence, but also globally consistent across the paragraph. This distinction matters because many failures look acceptable in isolation. A good example is keeping a narrator as first person across the full passage instead of accidentally switching to third person in one line.
  * **A good trajectory contains at least one resolved ambiguity.** There should be a moment where the agent chooses among two locally plausible translations by consulting wider context. This creates diagnostic value for the example. If no ambiguity is present, the instance may be too easy for this skill.
  * **A good trajectory ends in a document-level translation, not a sentence pile.** The final output should read like a connected passage with stable voice and reference structure. This is what differentiates the skill from sentence-level MT. If the output feels stitched together from independent local choices, it should fail selection.
