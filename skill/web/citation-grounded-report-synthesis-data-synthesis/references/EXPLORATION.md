# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - Start with breadth collection before depth writing. The agent should gather enough sources to cover the major subtopics of the report, then identify which ones are suitable for grounding central claims and which ones are merely contextual. As an edge case, if all early sources cluster around one viewpoint, the agent should deliberately search for a second perspective before drafting.
  - Extract candidate claim-to-source pairs as early as possible. Instead of browsing first and citing later, the agent should convert each useful page into one or more supportable statements linked to a URL or document identifier. This prevents the common failure mode where the final report is fluent but only weakly grounded.
  - Build a report outline from evidential coverage gaps, not from generic writing habits. Each section should correspond to a user need and a supported evidence cluster such as problem framing, methods, trade-offs, or recommendations. If a needed section lacks evidence, the agent should return to exploration before writing.
  - Track both coverage and trustworthiness during collection. A source can be relevant enough to mention but too weak to support a central conclusion, so the agent should decide which pages ground core claims and which only provide auxiliary context. For example, a blog post may help identify terminology, while a technical report or official documentation should anchor a stronger recommendation.

* **Target Trajectory Profile**:
  - A target trajectory should include multiple sources mapped to distinct claims or sections. The path should make clear that the final report is assembled from a coverage plan rather than from one catch-all page. If one source dominates every major assertion, the item is underpowered for citation-grounded synthesis.
  - The trajectory should include an explicit evidence-to-outline transition. Before generating the final report, the agent should show how retrieved evidence clusters correspond to the structure of the answer. This is important because grounded synthesis is as much about organizing evidence as about collecting it.
  - A strong trajectory preserves source granularity. The agent should know which source supports which sentence-level or section-level idea instead of attaching the same citation everywhere. As an edge case, broad survey pages should not be used to justify precise numerical or methodological claims unless they truly contain them.
  - The final output should remain answer-focused while still evidentially disciplined. The report should not devolve into a bibliography dump or a generic literature survey. It should answer the user’s actual problem using claims that can each be traced back to collected evidence.
