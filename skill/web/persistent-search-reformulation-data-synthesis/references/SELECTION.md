# Phase 2: Trajectory Selection Criteria
* **Acceptance Metrics**:
  - Accept the trajectory only if it contains at least three semantically distinct search formulations. Each formulation must be motivated by a different clue emphasis, document target, or candidate-reduction strategy. If the trajectory merely adds or removes one token at a time, it does not meet the bar.
  - Accept only when the agent documents a visible narrowing process from many possibilities to one. There should be an intermediate stage where at least two plausible candidates are considered and then reduced using newly retrieved evidence. As an example, a candidate publication list is acceptable only if the later steps explain why the non-winning papers fail the educational constraints.
  - Accept only if at least one failed or low-quality branch is identified and corrected. This requirement ensures the data reflects persistence rather than linear luck. If the trajectory is perfectly straight with no evidence of course correction, reject it as too easy for this capability.
  - Accept only if the final answer is supported by evidence obtained after one or more reformulations. The decisive page must not be directly recoverable from the first query without meaningful search evolution. As an edge case, if the first query lands on the answer page by chance, the example should be discarded or rewritten.

* **Rejection Criteria**:
  - Reject any trajectory where the first query or first clicked result directly reveals the answer. Such a path tests recall, not reformulation skill. Even if the final answer is correct, the trajectory is too trivial for this capability.
  - Reject trajectories built on repetitive keyword fiddling with no change in retrieval strategy. Slight edits like pluralization, punctuation, or moving words around do not demonstrate search planning. The example should only survive if the agent changes how it conceptualizes the problem.
  - Reject trajectories that never produce uncertainty. This capability requires a genuine search space where the agent must decide how to pivot, so a path with no ambiguity, no branching, and no candidate management is out of scope. For example, a single official page with the exact question wording is an automatic reject.
  - Reject trajectories that depend on non-web shortcuts such as hidden annotations, pre-exposed candidate lists, or privileged APIs unavailable to a normal web agent. The point is to test open-web persistence under ordinary browsing conditions. If the path would not generalize to a browser-only agent, it should not be synthesized.
