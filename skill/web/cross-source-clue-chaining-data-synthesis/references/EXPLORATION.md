# Phase 1: Environment Exploration Guide
* **Exploration Strategy**:
  - Start by decomposing the question into explicit clue slots before opening pages. Each slot should correspond to a fact type such as identity, time, affiliation, numeric property, or membership relation, and the agent should search for pages that can fill one slot at a time. As an example, the NASA question naturally decomposes into image identity, astronaut group, group membership list, and mission-duration comparison.
  - Collect evidence from heterogeneous page types on purpose. A good exploration path mixes overview pages, structured rosters, biography pages, and record pages because clue-chaining tasks are designed so that no one genre is sufficient. If all evidence comes from one page template or one host, the item is likely too centralized for this capability.
  - Maintain a living candidate table while browsing. After each page, update which clue slots are resolved, which candidate entities remain alive, and which relation still needs confirmation. As an edge case, if two candidates share the same last name or affiliation, the table should force the agent to capture a second identifier before moving on.
  - Prefer follow-up moves that bridge unresolved slots instead of gathering redundant detail. Once a page resolves one clue, the next action should target the next missing relation in the chain rather than deepening an already-settled point. For example, after confirming an author affiliation, the agent should move to author education or publication metadata rather than collecting five more affiliation pages.

* **Target Trajectory Profile**:
  - A target trajectory should show at least two independent evidence sources contributing non-redundant facts. Each source must resolve a different clue slot that is necessary for the final answer. If one page alone could have answered the question, the trajectory is not a true clue-chaining example.
  - The trajectory should preserve entity continuity across hops. The agent must make clear why the person, paper, team, or event found on one page is the same object referenced on the next page. An edge case is name collision; when names are common, the trajectory should include an additional bridge such as institution, date, or role.
  - A strong trajectory includes an explicit intermediate narrowing stage. Before the final answer is produced, the agent should hold a partial but incomplete hypothesis and then verify the remaining link. For instance, identifying the likely publication is not enough until the author background pages confirm the remaining constraints.
  - The end of the trajectory should culminate in a synthesis step, not a single-page extraction step. The final answer should only be derivable after the agent combines the resolved clue slots into a unified conclusion. If the last page simply states the answer with all earlier hops unnecessary, the example is overfit and should be rejected.
